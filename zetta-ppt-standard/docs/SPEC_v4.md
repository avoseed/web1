## 0. 결론 (So What)

- 규격·팔레트·표헤더·크롬 좌표는 **v3 실측값과 v4 규칙이 대부분 일치** → 그대로 정본 채택.
- **단 1건 교정**: 폰트 latin 지정. v3 테마는 `Arial`, v4 규칙은 **맑은고딕 3중지정(latin·ea·cs)** → 코드에서 3중 강제.
- 원형 6종(표지·간지·P본문·F본문·재무표·종료)으로 Ocado 정기협의체 덱 전 페이지 커버 가능.

### 2-1. 수직 간격 표준 (v4.1, 정기협의체 산출물 실측)

개체 간 간격은 아래 토큰으로 **엄격 고정** — 임의 간격 금지.

| 토큰 | 값 (cm) | 의미 |
|---|---|---|
| 헤더 하단 → 리드메시지 | **0.80** | 제목 블록(~2.10)과 리드(2.90) 사이 여백 |
| 리드 하단 → 본문 시작 `BODY_TOP_LEAD` | **0.55** (T=4.10) | 리드(~3.55)와 첫 본문 블록 사이 여백 |
| 본문 블록 간 `GAP_SECTION` | **0.50** | 소제목 묶음(불릿·표) 종료 → 다음 소제목 |
| 소제목 피치 `GAP_HEAD` | **0.55** | 소제목 시작 → 첫 본문 요소 시작 |
| 개조식 불릿 행간 `LINE_H` | **0.48** | 불릿 텍스트박스 수직 피치 |

### 2-2. 개조식 불릿 체계 (v4.1, 정기협의체 산출물 실측)

| 수준 | 마크 | 서식 | 용도 |
|---|---|---|---|
| 리드 | **■** | 14pt 볼드 검정 (크롬 `add_lead`) | 페이지 핵심 한 줄 |
| 주항목 | **•** | 볼드 검정 | 블록 제목·핵심 항목 (`add_bullets` level 0) |
| 하위 | **-** | 일반, +0.55cm 들여쓰기 | 상세 설명 (level 1) |
| 세부 | **·** | 일반, +1.10cm 들여쓰기 | 부가 설명 (level 2) |

> `□`·`○` 등 이형 마크 **사용 금지**. 단위·기준 표기(`(단위 : 억원, %)`)는 수치 표가 있을 때만 — **날짜형 기준 표기 금지**.

> v3→v4 변경분: 제목 가운데→**좌측 브래킷**, 페이지번호 하단중앙→**우하단**, 챕터간지 레드로고→**회색 풀블리드**.

## 3. 슬라이드 원형 레지스트리

| ID | 원형 | 용도 | 헤더 tier | 핵심 슬롯 |
|---|---|---|---|---|
| **C** | 표지 | 덱 표지 | — | 태그(내부 토의용)·대제목(밑줄)·조직·일자 |
| **D** | 챕터 간지 | 섹션 구분 | — | 회색 풀블리드·`N. 제목`·①②③ 하위항목 |
| **P** | P계열 본문 | 경영실적·현안·전략(본편) | P(검정 28pt) | 자유본문 + 재무표/차트 |
| **F** | F계열 본문 | Back-up·Link·참고자료 | F(네이비 20pt) | 자유본문 + 표/이미지 |
| **T** | 재무형 표 | 손익·투자·KPI | (P/F 위에 얹음) | EAEEF6 헤더·1열 좌측·숫자 우측 |
| **Z** | 종료 | 마감 | — | No.1 GROCERY MARKET·RE:TARGET/DESIGN/FOCUS |

### 재무형 표(T) 규칙
- 헤더행: **#EAEEF6**(1차)/#D8E0EC(2차) 채움 + **네이비 볼드 + 가운데정렬**.
- 1열(구분): **좌측정렬 볼드**. 2열~: **우측정렬**(숫자). 서술형 열은 좌측정렬 허용.
- **격자: 전 셀 4방 #BFBFBF 0.75pt 실선을 코드로 명시**(v4.1 — 테마 기본선에 의존 금지), 밴딩 off, 셀 상하 여백 0·좌우 0.1cm, 세로 가운데.
- 감소값 `▲` 접두(개조식 관례 유지).

## 4. v3 실측 ↔ v4 규칙 정합성 검증

| 항목 | v3 실측 | v4 규칙 | 판정 | 처리 |
|---|---|---|---|---|
| 페이지 규격 | 27.52×19.05 A4 | 27.52×19.05 | **사실** | 채택 |
| 폰트 ea | 맑은고딕 | 맑은고딕 | **사실** | 채택 |
| 폰트 latin | Arial (테마) | 맑은고딕 3중지정 | **오류** | **교정**(코드 3중 강제) |
| 브래킷 헤더 | `[내부토의양식①] 제목` | `[분류①] 제목` 좌상단 | **사실**(형식 동일) | 채택 |
| 제목 정렬 | 좌측 | 좌측 브래킷 | **사실** | 채택 |
| 페이지번호 | 우하단 `0/0`(n/N) | 우하단 n/N | **사실** | 채택 |
| 단위 표기 | 우상단 | 우상단 | **사실** | 채택 |
| 각주 | 좌하단 ※ | 좌하단 각주 | **사실** | 채택 |
| 표 헤더색 | EAEEF6/D8E0EC | #EAEEF6/#D8E0EC | **사실** | 채택 |
| 네이비/크림슨 | 000066 / C30C3E | #000066 / #C30C3E | **사실** | 채택 |
| 간지 배경 | bg1 lumMod95%(회색) | 회색 풀블리드 | **사실** | 채택 |
| 제목 크기 | 20pt 단일 | P 28~32 / F 20 | **부분사실** | ~~v4 이원화~~ → **v4.1 실측 회귀: P/F 공통 20pt** (tier 는 색상만 구분) |
| 표 격자 | #BFBFBF 실선 | 격자 #BFBFBF | **사실** | v4.1 코드 강제(전 셀 4방 0.75pt) — 기존 빌더 누락 교정 |

> `[확인] …`(빨강)·`XXXX`·`※ 샘플양식`은 **가이드 요소 → 최종본 삭제 대상**. 빌더는 미생성.

## 5. 호출 문법 (프롬프트)

빌더 모듈 사용 시 아래 압축 지시로 표준 덱 생성:

```
new_deck()
add_cover(prs, "제목", org="온라인사업단", date="2026. 07. 06", tag="내부 토의용")
add_divider(prs, 1, "섹션제목", ["하위①","하위②","하위③"])
s = add_content(prs, "경영실적①", "제목", n, total, tier="P", lead="핵심 한 줄 메시지", unit="(단위 : 억원, %)", footnote="※ ...")   # lead 는 `■ ` 자동 접두
add_fin_table(s, MARGIN_L, BODY_TOP, 25.26, 8.0, data, col_w=[...], header_rows=1)
s = add_content(prs, "Back-up", "제목", n, total, tier="F")   # F계열
add_closing(prs)
prs.save("out.pptx")
```

압축 지시 예: **"표지 + 간지1 + P(경영실적①, 표) + F(Back-up) + 종료"** → 위 시퀀스로 매핑.

## 6. 산출 계약 (v4)

1. **명세서**(본 문서) — 토큰·원형·검증표
2. **완결형 코드** `zetta_ppt_standard.py` — 3중폰트·크롬·원형·재무표 함수
3. **시각검수** — LibreOffice PDF→pdftoppm 렌더 후 오버플로/정합 점검

### 금지·문체 규칙 (유효 유지)
- 개조식 명사형 종결, 영문 약어 최소화, 단정적 표현.
- 제목 underline·decorative bar 남용 금지(간지/표지 밑줄은 표준 요소로 예외).
- P계열 제목은 1행 권장(≤ 약 24자); 초과 시 헤더 H 상향 또는 분리.
