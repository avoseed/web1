## 0. 결론 (So What)

- 규격·팔레트·표헤더·크롬 좌표는 **v3 실측값과 v4 규칙이 대부분 일치** → 그대로 정본 채택.
- **단 1건 교정**: 폰트 latin 지정. v3 테마는 `Arial`, v4 규칙은 **맑은고딕 3중지정(latin·ea·cs)** → 코드에서 3중 강제.
- 원형 6종(표지·간지·P본문·F본문·재무표·종료)으로 Ocado 정기협의체 덱 전 페이지 커버 가능.

### 2-1. 수직 간격 표준 (v4.1, 정기협의체 산출물 실측)

개체 간 간격은 아래 토큰으로 **엄격 고정** — 임의 간격 금지.

| 토큰 | 값 (cm) | 의미 |
|---|---|---|
| 헤더 하단 → 리드메시지 | **0.80** | 제목 블록(~2.10)과 리드(2.90) 사이 여백 |
| 리드 하단 → 본문 시작 `BODY_TOP_LEAD` | **0.55** (T=4.10) | 리드(~3.55)와 첫 본문 블록 사이 여백 |
| 본문 블록 간 `GAP_SECTION` | **0.50** | 소제목 묶음(불릿·표) 종료 → 다음 소제목 |
| 소제목 피치 `GAP_HEAD` | **0.55** | 소제목 시작 → 첫 본문 요소 시작 |
| 개조식 불릿 행간 `LINE_H` | **0.48** | 불릿 텍스트박스 수직 피치 |

### 2-2. 개조식 불릿 체계 (v4.1, 정기협의체 산출물 실측)

| 수준 | 마크 | 서식 | 용도 |
|---|---|---|---|
| 리드 | **■** | 14pt 볼드 검정 (크롬 `add_lead`) | 페이지 핵심 한 줄 |
| 주항목 | **•** | 볼드 검정 | 블록 제목·핵심 항목 (`add_bullets` level 0) |
| 하위 | **-** | 일반, +0.55cm 들여쓰기 | 상세 설명 (level 1) |
| 세부 | **·** | 일반, +1.10cm 들여쓰기 | 부가 설명 (level 2) |

> `□`·`○` 등 이형 마크 **사용 금지**. 단위·기준 표기(`(단위 : 억원, %)`)는 수치 표가 있을 때만 — **날짜형 기준 표기 금지**.

> v3→v4 변경분: 제목 가운데→**좌측 브래킷**, 챕터간지 레드로고→**회색 풀블리드**. **v4.1: 페이지번호 크롬 제거.**

### 3-1. 본문 프레임워크 유형 체계 (v4.1 — 정기협의체 전 장표 + OSP 경과보고 실측)

**단순 상하 나열식 구성 금지** — 본문은 아래 유형 중 택일(혼합 가능). 리드메시지 `■`는 전 유형 공통(요약 S형은 본문 자체가 ■ 나열).

| 유형 | 실측 근거 | 구조 |
|---|---|---|
| **S. 보고 요약** | 금일 보고 요약 | `■` 볼드 요약문 3~5행만 넉넉한 행간으로 나열 — 덱 서두 |
| **A. 병렬 2단 컬럼** | 상품(8/15) · 마트/슈퍼(14/15) · CFC 포장(Link) | **컬럼 헤더 바**(#EAEEF6 네이비 볼드) 또는 **【 】 블록 소제목** + **중앙 세로선** → 컬럼 내 `•` 불릿·표 → (선택) 하단 **네이비 결론 박스** |
| **B. 근거 표/차트 + 해설** | 운영/서비스(12/15) · 8월 Capa(Link) | 상단 **전폭 표·차트**(근거 수치) → 하단 2~3단 분해(밑줄 단제목 or 【 】 + `•` 불릿, 보조 표) |
| **C. 재무 전폭 표** | 경영실적①~③(1·3·4/15) | 전폭 재무표 = 본문. 필요 시 우측 **비고 패널**(①② 주석·미니표) |
| **D. 구분 매트릭스** | 신선상품 포장 기준(Link) · 고객 경험(Link) | 좌측 **구분 라벨**(플레인 볼드) + **가로 괘선** 행 구분 + 우측 상세(【 】 블록·불릿·표·이미지) |
| **E. 경과/타임라인** | OSP E2E 경과 · Coles 협업(Link) · 새벽배송 포장(Link) | 상단 **가로 타임라인**(굵은 축 + 마일스톤 날짜/라벨) → 하단 분석 블록(2단·빨간 밑줄 소제목) → **`→ 결론 :` 한 줄** 또는 **시사점 박스** |

- 강조 장치: **크림슨 밑줄**(핵심 수치·문구), **네이비 결론 박스**(So-What), **시사점 박스**(회색 테두리 + 짙은 탭 라벨), `→`(인과·결론).
- 단계 시리즈 장표는 `Phase N. 문장 + 밑줄 룰` 리드 변형 허용(5·6/15).
- **마케팅 장표형 회색 레이블 레일(7/15·Value Proposition 류)·키메시지/단계 박스 프레임워크 사용 금지** — 본장 원형이 아님.
- 빌더 원형: `add_col_header` `add_vsep` `add_conclusion_box` `add_block_header` `add_hline` `add_timeline` `add_insight_box` + `COL_GAP`/`COL_HEADER_H`/`CONCL_H` 토큰.

### 재무형 표(T) 규칙
- 헤더행: **#EAEEF6**(1차)/#D8E0EC(2차) 채움 + **네이비 볼드 + 가운데정렬**.
- **정렬(v4.1, 정기협의체 본장 실측): 헤더·본문 전 셀 가운데 고정.** 재무 수치열만 `col_align`(예: `["c","r","r"]`)으로 우측 지정. **셀별 임의 정렬 금지.** 1열(구분)은 볼드.
- **격자: 전 셀 4방 #BFBFBF 0.75pt 실선을 코드로 명시**(v4.1 — 테마 기본선에 의존 금지), 밴딩 off, 셀 상하 여백 0·좌우 0.1cm, 세로 가운데.
- 감소값 `▲` 접두(개조식 관례 유지).

## 5. 호출 문법 (프롬프트)

빌더 모듈 사용 시 아래 압축 지시로 표준 덱 생성:

```
new_deck()
add_cover(prs, "제목", org="온라인사업단", date="2026. 07. 06", tag="내부 토의용")
add_divider(prs, 1, "섹션제목", ["하위①","하위②","하위③"])
s = add_content(prs, "경영실적①", "제목", tier="P", lead="핵심 한 줄 메시지", unit="(단위 : 억원, %)", footnote="※ ...")   # lead 는 `■ ` 자동 접두
add_fin_table(s, MARGIN_L, BODY_TOP, BODY_W, 8.0, data, col_w=[...], header_rows=1, col_align=["c","r","r"])   # 수치열만 'r'
s = add_content(prs, "Back-up", "제목", tier="F")   # F계열
add_closing(prs)
prs.save("out.pptx")
```

압축 지시 예: **"표지 + 간지1 + P(경영실적①, 표) + F(Back-up) + 종료"** → 위 시퀀스로 매핑.

## 6. 산출 계약 (v4)

1. **명세서**(본 문서) — 토큰·원형·검증표
2. **완결형 코드** `zetta_ppt_standard.py` — 3중폰트·크롬·원형·재무표 함수
3. **시각검수** — LibreOffice PDF→pdftoppm 렌더 후 오버플로/정합 점검

### 금지·문체 규칙 (유효 유지)
- 개조식 명사형 종결, 영문 약어 최소화, 단정적 표현.
- **글밥 최소화 (v4.1)**: 불릿은 1행 이내 핵심 명사구, 조사·수식어·중복 정보 배제. 상세 수치·조건은 표·각주로 이관. 핵심 내용은 빠짐없이 담되 문장으로 풀지 않는다.
- 제목 underline·decorative bar 남용 금지(간지/표지 밑줄은 표준 요소로 예외).
- P계열 제목은 1행 권장(≤ 약 24자); 초과 시 헤더 H 상향 또는 분리.
